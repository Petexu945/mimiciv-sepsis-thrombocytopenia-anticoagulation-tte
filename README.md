# MIMIC-IV Sepsis Thrombocytopenia Anticoagulation Target Trial Emulation

This repository contains the core SQL and R code used for the study:

**Early prophylactic anticoagulation in patients with severe sepsis-associated thrombocytopenia: a target trial emulation**

The study emulates a target trial comparing early pharmacological venous thromboembolism prophylaxis versus no early pharmacological prophylaxis among critically ill adults with severe sepsis-associated thrombocytopenia, defined by platelet counts of 30,000–50,000/µL.

## Repository contents

```text
sql/
  01_build_primary_cohort.sql

R/
  01_main_analysis.R
```

## Purpose of the code

`sql/01_build_primary_cohort.sql` constructs the primary analysis cohort from MIMIC-IV. The script includes the main steps for cohort construction, platelet-defined time zero, Sepsis-3 eligibility, exclusion criteria, 24-hour grace-period exposure assignment, baseline covariate extraction, outcome derivation, and construction of the final analysis table.

`R/01_main_analysis.R` performs the main statistical analyses, including propensity-score estimation, inverse probability weighting, covariate balance assessment, primary weighted Cox models, entropy balancing, propensity-score matching, secondary outcome analyses, net clinical benefit estimation, subgroup analyses, and sensitivity analyses.

## Data availability

This repository contains code only.

No patient-level MIMIC-IV data, derived datasets, intermediate tables, analysis-ready datasets, database extracts, or model output files are included in this repository.

The study uses MIMIC-IV data, which are available from PhysioNet to credentialed users who have completed the required training and data use agreement. Users who wish to reproduce the analysis must obtain independent authorized access to MIMIC-IV and run the code within their own credentialed environment.

## Database configuration

Database credentials should not be written directly into the analysis script. The R script reads database information from environment variables.

Before running the R script, users should configure the following environment variables in their local environment:

```text
MIMIC_DBNAME
MIMIC_HOST
MIMIC_PORT
MIMIC_USER
MIMIC_PASSWORD
MIMIC_ANALYSIS_SCHEMA
```

The default analysis schema is assumed to contain the SQL-derived analysis tables generated by `01_build_primary_cohort.sql`.

## Analysis workflow

The intended workflow is:

1. Load MIMIC-IV into a local PostgreSQL database.
2. Run `sql/01_build_primary_cohort.sql` to construct the primary analysis cohort and analysis-ready SQL table.
3. Configure the required database environment variables.
4. Run `R/01_main_analysis.R` to reproduce the primary and sensitivity analyses.

## Important notes

This repository is intended to support transparency and reproducibility of the core cohort construction and statistical analysis code.

Because MIMIC-IV is a restricted-access database, the repository does not include any patient-level data or derived datasets.

Users are responsible for ensuring that their local MIMIC-IV schema names and database configuration match the script settings before execution.
